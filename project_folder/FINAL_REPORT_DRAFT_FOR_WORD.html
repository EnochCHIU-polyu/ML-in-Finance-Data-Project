Multi-modal S&P 500 Forecasting with Cluster-Aware Sentiment

Academic Report Draft | Role: Senior Quant Research & Machine Learning Engineer

Abstract

This study develops a multi-modal forecasting system for S&P 500 stocks by integrating technical signals, sentiment signals, and unsupervised clustering structure. The project addresses two coupled tasks: Task A performs next-day close prediction, while Task B builds sector/correlation-aware clusters that expand and denoise sparse ticker-level news.

Model benchmarking across linear models, tree models, and deeper sequence models indicates that XGBRegressor is the most reliable final predictor under the current data regime. Deeper models (CNN/LSTM/GRU-style candidates) show weaker generalization due to limited effective sample size per ticker after temporal splitting, and higher sensitivity to regime non-stationarity.

A key breakthrough is the use of cluster-derived peer sentiment as a collective intelligence mechanism for zero-news days. Using archived result tables, the Sentiment-Enhanced Blended Strategy reaches MAE = 4.8383, RMSE = 9.2974, R2 = 0.999541, while the Dynamic Cluster-News Stacked Strategy reaches MAE = 4.5109, RMSE = 8.8023, R2 = 0.999583 and Directional Accuracy = 0.5467. This confirms that deployment-grade model selection must be objective-specific rather than metric-singleton.

1. Introduction and Research Questions

Forecasting S&P 500 prices is difficult because financial time series are non-stationary, low-SNR, and regime-dependent. A model optimized for point-wise error may not maximize risk-adjusted return after execution constraints. This report therefore formalizes a dual-objective view.

Research Questions

  1. Can clustering transform sparse ticker-level news into a denser, usable sentiment signal?
  2. Why does XGBRegressor remain the final choice despite attempts with deeper architectures?
  3. How should sentiment-enhanced blended strategies versus dynamic cluster-news stacked strategies be interpreted under trading utility versus point-wise accuracy?

Experiment Naming Convention

  • Sentiment-Enhanced Blended Strategy: Global + cluster-specific sentiment fusion for trading utility.
  • Dynamic Cluster-News Stacked Strategy: Two-stage baseline + residual news stacking for fit optimization.
  • Cluster-Aware Baseline Comparison Model: Earlier cluster-aware regression benchmark used for same-spec comparison.

Core Thesis: Clustering is not only an unsupervised analysis module; it is a structural signal router that converts sparse direct-news streams into peer-informed predictive features.

2. Formulation and System Design

2.1 Problem Statement

Let i denote stock, t denote day. Task A predicts next-day price (or return-transformed target), while Task B learns cross-sectional stock structure for feature expansion.

Task A: ŷi,t+1 = f(Xi,t)
Task B: c(i) = KMeans(Zi)

where X includes technical, sentiment, and cluster-derived features; c(i) is the cluster ID of stock i from unsupervised learning over structural descriptors.

2.2 End-to-End Pipeline

  1. Market Data Ingestion: OHLCV from Kaggle S&P 500 source, normalized by ticker/date.
  2. News Ingestion: Multi-source finance headlines with deduplication and source tagging.
  3. Sentiment Encoding: Daily ticker sentiment generation from cleaned headlines.
  4. Temporal Feature Memory: lagged-decay sentiment using 10-day lookback and decay 0.2.
  5. Task B Clustering: K-Means over risk/return/volume descriptors to generate peer sets.
  6. Task A Regression: XGBRegressor with technical + sentiment + cluster interaction features.
  7. Dual Evaluation: point-wise metrics + portfolio backtest metrics.

2.3 Sentiment Decay Lagging (Critical)

Exponential temporal weighting prioritizes recency and suppresses stale news impact:

wk = exp(-0.2(k-1)) / Σm=1..10 exp(-0.2(m-1)), k = 1..10
Sdiri,t = Σk=1..10 wk · sdiri,t-k

This design imposes causality (t-k only), improves robustness to delayed reactions, and avoids forward leakage.

2.4 Mathematical Coupling: How Task B Feeds Task A

For each stock i, cluster membership c(i) defines a peer set P(i). Cluster sentiment is built from peers and merged into regression features:

Sclui,t = Σk=1..10 wk · (1/|P(i)|) Σj∈P(i) sj,t-k
Xi,t = [Xtechi,t, Sdiri,t, Sclui,t, Smktt, φ(c(i))]
ŷi,t+1 = fXGB(Xi,t)

Conceptually, “Cluster News Sentiment = Collective Intelligence of Peer Stocks.” This mechanism directly mitigates sparse direct-news windows and improves feature availability for less-covered tickers.

2.5 Why XGBRegressor Is the Final Choice

  1. Tabular Fit: Feature space is mixed-type, interaction-heavy, and nonlinear; gradient-boosted trees are naturally effective.
  2. Sample Efficiency: Deep sequence models need longer stable horizons; effective sample size per ticker is constrained after strict temporal split.
  3. Generalization Stability: Deep models show stronger train/validation divergence in this regime, indicating overfit pressure without enough data support.
  4. Empirical Reliability: XGB retains strong MAE/RMSE while maintaining robust behavior under regime shifts.

2.6 Leakage Control and Evaluation Discipline

  • All sentiment inputs are lagged before prediction day.
  • No random split; only time-ordered splitting for realistic forward validation.
  • Separate chart-level diagnostics (single stock) from portfolio-level execution tests (cross-stock daily aggregation).

3. Innovation and Improvements

3.1 Evidence-Driven Temporal Sentiment Memory (Auto-Tuned Lag+Decay)

Instead of manually setting temporal sentiment memory, this project introduces an auto-tuned lag/decay mechanism with grid-search evidence. The selected configuration (10-day lookback, decay 0.2) is not a heuristic guess; it is chosen from a stable low-MAE region and then propagated to regenerated comparison tables and ranking outputs. This turns sentiment memory design into a reproducible model-selection process.

Figure S5 — Lag-Decay Grid Search Surface
Caption: MAE heatmap over lookback-days and decay-rate combinations.
Deep Analysis: The nearly flat optimum indicates robustness rather than fragile hyperparameter luck. The chosen setting (10-day, decay 0.2) sits in a stable low-MAE zone and is suitable for reproducible deployment.

3.2 Window-Consistent Sentiment Pipeline Refinement

Beyond adding sentiment features, the sentiment-enhanced blended workflow is refined around a consistent 10-day sliding-window design and synchronized cache/output behavior. Notebook logic and generated artifacts are updated together, reducing temporal inconsistency between feature generation and regression evaluation. This improves reproducibility and makes cross-run comparisons more defensible.

3.3 New Dynamic Cluster-News Pathway

A dedicated dynamic cluster-news experiment path is added as an architectural extension rather than a minor retune of the sentiment-enhanced blended workflow. The design combines clustering-driven news selection with XGBoost and integrates full-universe news ingestion plus shared caches. This creates a second, clearly defined modeling pathway for testing whether dynamic cluster-news stacking can outperform baseline formulations under a fit-oriented objective.

3.4 Evaluation Protocol Correction for Deployment Credibility

The evaluation design is corrected by separating single-stock trajectory diagnostics from daily portfolio-level backtesting. This avoids mixing chart-level fit inspection with execution-level utility assessment and strengthens methodological credibility. Under the revised protocol, the stacked cluster-news approach shows stronger baseline-relative performance in updated comparisons.

The comparison is objective-specific, not contradictory. In the dynamic cluster-news evaluation, the stacked cluster-news model improves point-fit and direction metrics at the same time: MAE falls from 4.6832 to 4.5109 and Directional Accuracy rises from 0.5167 to 0.5467 versus baseline XGB. This supports using stacked cluster-news when the objective emphasizes error minimization with stable directional behavior.

5. Conclusion and Future Work

This project demonstrates that structural coupling between unsupervised clustering and supervised forecasting can produce practical alpha extraction under noisy financial conditions. The key contribution is converting sparse individual-news streams into cluster-informed sentiment signals that improve robustness in real-world ticker coverage.

Methodologically, the report supports a dual-objective governance framework: use MAE/RMSE/R² to monitor fit quality and direction-related metrics to assess execution usefulness. Under this lens, the sentiment-enhanced blended strategy records strong aggregate fit (MAE = 4.8383, RMSE = 9.2974), while the dynamic cluster-news stacked strategy reaches the best absolute fit in current outputs (MAE = 4.5109, RMSE = 8.8023).

Future Work

  • Latency-aware sentiment alignment to handle delayed market reaction windows.
  • Regime-adaptive decay rather than fixed decay coefficient.
  • Dynamic/online clustering for evolving sector correlation topology.
  • Uncertainty-aware prediction intervals for position sizing and risk budget control.
