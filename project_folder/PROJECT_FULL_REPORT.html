This project builds a complete multi-modal machine learning system for S&P 500 stocks, combining four tightly integrated tasks: trend classification (Task 01), price regression (Task 02), clustering analysis (Task 03), and anomaly detection (Task 04). 02D — ClusterNews Stacked XGBoost (Current Best)

The key innovation of 02D is the two-stage stacked architecture:

  1. Stage 1 (Price Baseline): XGBoost trained on 24 technical indicators → produces price predictions and residuals
  2. Stage 2 (News Correction): Second XGBoost trained to predict Stage 1 residuals using cluster-aggregated sentiment features → corrects systematic bias
  3. Final Prediction: Stage 1 prediction + Stage 2 residual correction

Cluster news aggregation: for each target stock, sentiment scores from all peer stocks within the same K-Means cluster are pooled (equal-weight average). This dramatically increases news coverage from ~40% (target stock alone) to ~95% (full cluster).

Focus 1: Cluster News Method — Deep Analysis

Research Question How should sparse financial news be incorporated into a stock price prediction model, and does clustering peer stocks improve the quality of the sentiment signal?

4.1 The News Sparsity Problem

Individual stocks receive news sporadically — on average 40–60% of trading days have no relevant headline for a given ticker. Using raw per-stock sentiment creates a sparse feature that degrades model training (most rows show sentiment=0). The cluster news approach resolves this by pooling sentiment across correlated peer stocks.

4.2 Three-Level Sentiment Strategy Comparison

Level 1

Direct Stock Sentiment

Only news about the target ticker. Coverage ~40-60%. Creates many zero-sentiment days that add noise.

Problem
High sparsity; model ignores sentiment on empty days.
Level 2

Global Sentiment (02C)

Average FinBERT score across all S&P 500 news in a given day. Always non-zero. Market-wide sentiment proxy.

Problem
Blends sector-specific signal with market noise.
Level 3

Cluster-Aggregated Sentiment (02D) ✓

Weighted average sentiment of all stocks in the same K-Means return-correlation cluster. Peers share similar price drivers.

Advantage
High coverage (~95%), sector-relevant signal, peer-amplified sentiment.

4.6 Why Stacking Beats Direct Integration

Direct integration places sentiment features alongside technical features in a single model. This forces the model to simultaneously learn price momentum (a low-variance, high-signal feature) and sentiment dynamics (a high-variance, low-signal feature). The strong technical features dominate feature importance, effectively suppressing the sentiment signal.

Stacking separates the concerns: Stage 1 learns the price momentum signal to the best possible accuracy, and Stage 2 receives the unexplained residuals — precisely the portion of price movement that technical analysis cannot explain. News-driven sentiment is then the ideal feature to explain those residuals.

System Improvement Analysis

The following is a deep analysis of five structural weaknesses in the current system and concrete fixes to be implemented in the next development phase.

Issue 1

Lag/Decay Grid Search Plateau

Current Problem
All 15 (lookback × decay) combinations return identical MAE = 4.8484. Single train/val split is too short to differentiate lag effects.
Proposed Fix
Replace single split evaluation with TimeSeriesSplit(n_splits=5) walk-forward CV. Grid search on rolling 6-month windows to surface true temporal sensitivity.
Issue 2

Static Sentiment (No Velocity)

Current Problem
02C/02D use lagged raw FinBERT scores. A stock that moved from −0.3 to +0.3 sentiment over 10 days has the same feature value as one static at +0.3. Narrative momentum is invisible to the model.
Proposed Fix
Add sentiment_velocity_10d = (5-day rolling mean lag-1) − (5-day rolling mean lag-11). Add sentiment_acceleration as second derivative. Both are key new features in 02E.
Issue 3

NVDA Degradation under News

Current Problem
Stacked model is +1.5% worse than baseline for NVDA because NVDA's prices respond to discrete high-magnitude events (GTC, earnings), not gradual sentiment trends that the residual corrector was trained on.
Proposed Fix
Add per-stock "news sensitivity" weight = rolling correlation(sentiment_velocity, |return|) over 60 days. Multiply stacked correction by this weight. Low weight for NVDA on normal days; high weight on event-driven days.
Issue 4

T+1 Only (Highest Noise)

Current Problem
All models predict next-day absolute price — the hardest and noisiest horizon. The signal-to-noise ratio at T+1 is near the theoretical minimum; small model improvements require disproportionate engineering effort.
Proposed Fix
Build 02E to predict log returns at T+5, T+10, T+15 simultaneously using MultiOutputRegressor. Log returns are stationary and comparable across price levels. Target DHR ≥ 55% at T+15.
Issue 5

K-Means Uses Euclidean Distance

Current Problem
K-Means clustering is performed on return-correlation features using Euclidean distance. Financial peer networks should be defined by co-movement, not Euclidean proximity in feature space.
Proposed Fix
Use correlation distance matrix (d = 1 − |ρ|) with Affinity Propagation or OPTICS clustering. Alternatively: apply PCA whitening before K-Means to normalize the correlation structure.
Issue 6

No Anomaly Integration

Current Problem
Task 04 anomaly flags are not used by Task 02 models. Predictions made on Task-04-flagged anomaly days are likely much less accurate (NVDA: 6.38% anomaly rate), degrading overall metrics.
Proposed Fix
Add anomaly_votes (0–4) from Task 04 as a feature in Task 02, or use it as a trade-gate: suppress trading signal when anomaly_votes ≥ 3 to reduce risk during high-uncertainty periods.

Step 1: Target Engineering — Forward Log Returns

Log returns are chosen over raw prices because they are: (1) stationary (no integration order), (2) additive across time horizons (log(T+15) = sum of log daily returns), (3) approximately normally distributed, and (4) comparable across stocks with vastly different price levels (AAPL ~$175 vs AMZN ~$185).

# Forward Log Return targets (clipped at ±30% for outlier control) for h in [5, 10, 15]: df[f'fwd_logret_{h}d'] = np.log( df.groupby('ticker')['Close'].shift(-h) / df['Close'] ) df[f'fwd_logret_{h}d'] = df[f'fwd_logret_{h}d'].clip(-0.30, 0.30) # Drop rows where future data is unavailable (last 15 rows per ticker) df = df.dropna(subset=['fwd_logret_5d', 'fwd_logret_10d', 'fwd_logret_15d'])

02F: Regime-Filtered Trading Signal (Preview)

The ultimate goal of the 02E/02F pipeline is to combine short-term (T+1 from 02D) and medium-term (T+15 from 02E) signals into a single regime-filtered trade signal that suppresses false breakouts:

False Breakout Filter

If T+15 predicts negative log return but T+1 predicts up, the position weight is discounted by: w = f(|T+15_pred| × σ_stock). This prevents entering a long trade on a 1-day spike that the medium-term model expects to reverse.

VIX-Conditional Weighting

When VIX > 25, T+1 noise dominates and the model degrades. The meta-learner down-weights T+1 signals and amplifies T+15's trend prediction. Meta-feature: VIX × |T+1_pred − T+15_pred_norm| (signal disagreement under stress).

Sentiment Velocity Alignment

When sentiment_velocity_10d < 0 (falling narrative momentum) but T+1 predicts up (single positive headline), T+15's trend model discounts the T+1 signal because it sees through single-day noise via 10-day patterns.

Conclusion

This project demonstrates that a carefully engineered multi-modal ML system — combining return-correlation clustering, FinBERT sentiment analysis, and stacked residual learning — materially outperforms pure technical-analysis baselines on S&P 500 next-day price regression. Key conclusions:

Clustering is the Bridge

K-Means peer-group clustering is not just an analytical task — it is the infrastructure that makes sparse news data usable. Without clustering, news sentiment would be too sparse to contribute meaningfully to regression. K-Means (composite score 0.577) is the right choice for this pipeline due to full coverage and balanced cluster structure.

Stacking > Direct Integration

Directly appending sentiment features to a technical model suppresses them due to feature importance competition. Stacking — where Stage 2 corrects Stage 1 residuals using sentiment — explicitly targets the unexplained variance and improves both MAE (−3.7%) and directional accuracy (+3.0 pp).

T+1 is the Hardest Horizon

Despite a 0.99958 R² score (high absolute accuracy), directional accuracy of 54.67% is only marginally above random. This confirms the theoretical expectation: T+1 price prediction is near the efficiency frontier. The path forward is medium-term horizons (T+15) where the signal-to-noise ratio is fundamentally better.

Anomaly Detection as Risk Gate

NVDA's 6.38% consensus anomaly rate directly maps to its poor news-enhanced performance. Integrating Task 04 anomaly flags as a trade gate (suppress signals when anomaly votes ≥ 3) is a high-priority improvement that requires no new data or model training.
